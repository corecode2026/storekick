# 🎮 متجر Kick — دليل التشغيل الكامل (VS Code)

## 📁 هيكل المشروع
```
kick-store/
├── server.js              ← السيرفر الرئيسي
├── package.json
├── .env                   ← بياناتك الفعلية (لا تشاركه مع أي أحد!)
├── .env.example            ← نموذج فاضي للنسخ
├── data/
│   └── db.js               ← قاعدة بيانات بسيطة تُحفظ في ملف db.json
├── middleware/
│   └── adminAuth.js         ← حماية مسارات لوحة الإدارة
├── routes/
│   ├── auth.js              ← Kick OAuth الحقيقي (login → callback → session)
│   ├── store.js             ← المنتجات والشراء وإدارة الطلبات
│   └── points.js            ← النقاط، مكافأة المشاهدة، إضافة نقاط يدوياً
└── public/
    ├── index.html            ← الموقع كامل (الصفحات + لوحة الإدارة)
    ├── css/style.css
    └── js/app.js             ← كل منطق الواجهة + الربط بالـ API
```

---

## ⚡ التشغيل في VS Code

### 1. افتح المجلد في VS Code
File → Open Folder → اختار مجلد `kick-store`

### 2. افتح Terminal جوا VS Code
`Terminal` من الأعلى ← `New Terminal`

### 3. ثبّت الباكدجات
```bash
npm install
```

### 4. تأكد من ملف `.env`
لازم يكون فيه بيانات حقيقية (شوف القسم التالي لو ناقص شي).

### 5. شغّل السيرفر
```bash
npm start
```

افتح المتصفح على: **http://localhost:3000**

## 🔑 إعداد Kick OAuth (لازم قبل أي شي)

1. روح [kick.com/settings/developer](https://kick.com/settings/developer)
2. اعمل **Create Application** (أو استخدم تطبيقك الموجود)
3. تأكد إن **Redirect URI** هو بالضبط:
   ```
   http://localhost:3000/auth/callback
   ```
4. فعّل **"تفعيل الرابط عبر الويب"** (مهم جداً!)
5. خذ **Client ID** و **Client Secret**
6. عبّيهم في ملف `.env`:

```env
KICK_CLIENT_ID=...
KICK_CLIENT_SECRET=...
KICK_REDIRECT_URI=http://localhost:3000/auth/callback
SESSION_SECRET=أي_نص_طويل_عشوائي
ADMIN_KEY=أي_نص_سري_لحماية_الإدارة
PORT=3000
```

> ⚠️ **مهم:** لو سبق ونشرت الـ Client Secret في أي مكان (شات، فورم، إلخ)، اعمل **Regenerate** له من نفس صفحة Kick قبل النشر الفعلي.

## 🔐 لوحة الإدارة

- الرابط: نفس الموقع، زر **"⚙️ إدارة"** بالـ Nav
- بيانات الدخول الافتراضية: `admin` / `admin`
- تقدر تغيّرها من **الإعدادات → تغيير كلمة سر الإدارة** (تتغيّر بالواجهة فقط لحد ما تعيد تحميل الصفحة — للحفظ الدائم يحتاج تطوير إضافي)

### حماية API الإدارة
كل عمليات الإدارة (تعديل منتجات، حذف طلبات، إضافة نقاط) محمية بمفتاح `ADMIN_KEY`. هذا المفتاح موجود بملفين:
1. `.env` على السيرفر (`ADMIN_KEY=...`)
2. `public/js/app.js` بمتغير `ADMIN_KEY` بالأعلى

**لازم تكون نفس القيمة بالضبط بالملفين.**

---

## 🛒 تخصيص المنتجات

من **لوحة الإدارة → المنتجات** تقدر:
- تضيف منتج جديد (+ إضافة منتج)
- تعدّل أي منتج موجود (✏️ تعديل)
- ترفع صورة للمنتج
- توقف/تفعّل منتج بدون حذفه
- تحذف منتج نهائياً

المنتجات تُحفظ تلقائياً بملف `data/db.json`.

## ⭐ نظام النقاط

| الفعل | النقاط |
|-------|--------|
| نقاط ترحيبية (أول تسجيل دخول) | 100 |
| مكافأة كل ساعة مشاهدة | 50 |
| إضافة يدوية من الإدارة | حسب ما تحدد |

### إضافة نقاط يدوياً من لوحة الإدارة
من **الإدارة → المستخدمون** فيه خانة "إضافة نقاط يدوياً":
- اكتب اسم المستخدم على Kick
- اكتب عدد النقاط
- لو المستخدم مسجّل، النقاط تضاف فوراً
- لو لسا ما سجّل، النقاط تُحفظ وتُضاف تلقائياً أول ما يسجّل دخوله

## 💾 قاعدة البيانات

البيانات كلها (المستخدمين، الطلبات، المنتجات) تُحفظ بملف:
```
data/db.json
```
هذا الملف ينشأ تلقائياً أول مرة تشغّل السيرفر. **لا تحذفه** إلا إذا بدك تصفّر كل البيانات.

> ملاحظة: هذا حل بسيط مناسب للتشغيل المحلي أو مشروع صغير. لو حبيت قاعدة بيانات حقيقية (Firebase, MongoDB...) ممكن نطورها لاحقاً.

## 🚀 النشر على الإنترنت (مطلوب لتسجيل الدخول الحقيقي من أي مكان)

تسجيل الدخول بـ Kick OAuth **لازم يشتغل من رابط حقيقي على الإنترنت**، مش `localhost`. الخيارات:

### Railway (الأسهل، مجاني)
1. ارفع المشروع على GitHub
2. روح [railway.app](https://railway.app) → New Project → Deploy from GitHub
3. أضف متغيرات البيئة (نفس محتوى `.env`)
4. بعد النشر، حدّث:
   - `KICK_REDIRECT_URI` في `.env` على Railway لرابط مثل: `https://yourapp.up.railway.app/auth/callback`
   - **Redirect URI** بنفس القيمة في إعدادات Kick Developer

### Render
نفس الخطوات تقريباً، فيه خطة مجانية أيضاً.

## 📞 API Endpoints

| Method | URL | الوصف | محمي بـ |
|--------|-----|-------|---------|
| GET | `/auth/login` | بدء تسجيل الدخول بـ Kick | - |
| GET | `/auth/callback` | رجوع من Kick بعد الدخول | - |
| GET | `/auth/me` | بيانات المستخدم الحالي | جلسة المستخدم |
| POST | `/auth/logout` | تسجيل الخروج | جلسة المستخدم |
| GET | `/api/store/products` | قائمة المنتجات النشطة | - |
| POST | `/api/store/buy` | شراء منتج | جلسة المستخدم |
| GET | `/api/store/admin/products` | كل المنتجات (admin) | `x-admin-key` |
| POST/PUT/DELETE | `/api/store/admin/products/:id` | إدارة منتج | `x-admin-key` |
| GET | `/api/store/admin/orders` | كل الطلبات | `x-admin-key` |
| PUT/DELETE | `/api/store/admin/orders/:id` | إدارة طلب | `x-admin-key` |
| GET | `/api/points/history` | سجل نقاط المستخدم | جلسة المستخدم |
| POST | `/api/points/watch-reward` | مكافأة مشاهدة ساعة | جلسة المستخدم |
| POST | `/api/points/admin-add` | إضافة نقاط يدوياً | `x-admin-key` |
| GET | `/api/points/leaderboard` | لوحة المتصدرين | - |
